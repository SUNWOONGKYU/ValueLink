## 작업 방식

### Step 1: 기존 문서 분석

**읽어야 할 파일:**
```
Valuation_Company/
├── docs/ (문서 폴더 존재 시)
├── README.md (프로젝트 설명 존재 시)
└── (기타 사용자 가이드 문서)
```

**분석 항목:**
1. 기존 매뉴얼 구조
2. FAQ 항목
3. 설명 스타일 (전문적 vs 쉬운 언어)
4. 스크린샷 유무
5. 업데이트 빈도

### Step 2: 기존 문서 → Markdown 변환

**변환 가이드:**

| 기존 형식 | Markdown |
|----------|----------|
| 제목 (볼드체) | `# 제목`, `## 소제목` |
| 목록 (번호) | `1. 항목`, `2. 항목` |
| 목록 (점) | `- 항목`, `- 항목` |
| 코드/명령어 | `` `code` `` 또는 ``` |
| 링크 | `[텍스트](URL)` |

**주의사항:**
- Markdown 문법 준수
- 목차 구조화 (H1 → H2 → H3)
- 링크 상대 경로 사용 (`/projects/list`)

### Step 3: 개선 사항 적용

**목업의 문제점 식별 및 개선:**

```markdown
❌ 목업: 전문 용어 설명 없음
"DCF 평가를 신청하세요."

✅ 개선: 전문 용어 설명 추가
"DCF (현금흐름할인법, Discounted Cash Flow) 평가를 신청하세요.
기업이 미래에 창출할 현금흐름을 현재가치로 할인하여 기업가치를 평가하는 방법입니다."
```

```markdown
❌ 목업: 단계 설명 불명확
"프로젝트를 생성하세요."

✅ 개선: 단계별 설명 명확화
## 프로젝트 생성

### 1. 프로젝트 목록 페이지 접속

- 상단 메뉴 **내 프로젝트** 클릭
- 또는 마이페이지에서 **새 프로젝트** 버튼 클릭

### 2. 프로젝트 정보 입력

- **프로젝트명**: 예) ABC 스타트업 기업가치평가
- **평가 방법 선택**:
  - DCF (현금흐름할인법) - 800만원
  - Relative (상대가치평가) - 500만원
  - ...

### 3. 프로젝트 생성 완료

- **프로젝트 생성** 버튼 클릭
- 프로젝트 상세 페이지로 자동 이동
```

```markdown
❌ 목업: FAQ 답변 짧음
Q. 평가 비용은 얼마인가요?
A. DCF 800만원, Relative 500만원입니다.

✅ 개선: 상세한 답변
### Q6. 평가 비용은 얼마인가요?

평가 방법별 비용은 다음과 같습니다:

| 평가 방법 | 비용 | 설명 |
|----------|------|------|
| DCF (현금흐름할인법) | 800만원 | 안정적인 현금흐름 예측 가능 기업 |
| Relative (상대가치평가) | 500만원 | 유사기업이 많은 산업 |
| Asset (자산가치평가) | 600만원 | 자산 중심 기업 (부동산, 제조업) |
| Intrinsic (내재가치평가) | 600만원 | 성장률이 중요한 기업 |
| Tax (세법상평가) | 1,000만원 | 증여, 상속 등 세법상 평가 |

비용에는 회계사 직접 검토 및 1회 수정이 포함됩니다.
```

### Step 4: Best Practice 적용

**Markdown 문서화 패턴:**
- 목차 구조화
- 내부 링크 활용 (`[프로젝트 생성](#프로젝트-생성)`)
- 표 활용 (비교 정보)
- 코드 블록 사용 (계좌번호 등)

**검색 최적화:**
```markdown
# ValueLink 사용자 매뉴얼

## 목차

1. [시작하기](#시작하기)
   - [회원가입](#회원가입)
   - [로그인](#로그인)
2. [프로젝트 생성](#프로젝트-생성)
3. [평가 신청](#평가-신청)
   - [DCF 평가 신청](#dcf-평가-신청)
   - [Relative 평가 신청](#relative-평가-신청)
```
